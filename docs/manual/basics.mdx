本页提供 Mojo 语言的概览。

如果你了解 Python，那么很多 Mojo 代码看起来会很熟悉。但是，Mojo 融合了静态类型检查、内存安全、下一代编译器技术等特性。因此，Mojo 与 C++ 和 Rust 等语言也有很多共同点。

如果你更喜欢通过实践学习，请跟随 [Mojo 入门教程](/docs/manual/get-started)。

在本页中，我们将介绍 Mojo 的基本语法，让你能够快速开始编码并理解遇到的其他 Mojo 代码。Mojo 手册的后续章节将深入探讨这些主题，并在下面提供相应链接。

让我们开始吧！🔥

:::note

Mojo 是一门年轻的语言，还有许多[功能尚未实现](/docs/roadmap)。因此，Mojo 目前**不**适合初学者。即使是这个基础部分也假设你有一定的编程经验。不过，在整个 Mojo 手册中，我们尽量不假设你有任何特定语言的经验。

:::

## Hello world

以下是 Mojo 中的传统 "Hello world" 程序：

```mojo
def main():
    print("Hello, world!")
```

每个 Mojo 程序都必须包含一个名为 `main()` 的函数作为入口点。我们稍后会详细讨论函数，现在只需要知道你可以写 `def main():` 后跟缩进的函数体即可。

`print()` 语句的作用如你所料，将其参数打印到标准输出。

## 变量

在 Mojo 中，你可以通过简单地给新命名变量赋值来声明变量：

```mojo
def main():
    x = 10
    y = x * x
    print(y)
```

你也可以使用 `var` 关键字_显式_声明变量：

```mojo
var x = 10
```

使用 `var` 声明变量时，你还可以声明变量类型，可以有也可以没有初始值：

```mojo
def main():
    var x: Int = 10
    var sum: Int
    sum = x + x
```

隐式声明和显式声明的变量都是静态类型的：也就是说，类型在编译时确定，在运行时不会改变。如果你没有指定类型，Mojo 会使用第一次赋给变量的值的类型。

```mojo
x = 10
x = "Foo" # 错误：无法将 "StringLiteral" 值转换为 "Int"
```

更多详情请参阅[变量](/docs/manual/variables/)页面。

## 代码块和语句

函数、条件和循环等代码块使用冒号后跟缩进行来定义。例如：

```mojo
def loop():
    for x in range(5):
        if x % 2 == 0:
            print(x)
```

你可以使用任意数量的空格或制表符进行缩进（我们推荐使用 4 个空格）。

Mojo 中所有代码语句都以换行符结束。但是，如果你缩进后续行，语句可以跨越多行。例如，这个长字符串跨越两行：

```mojo
def print_line():
    long_text = "这是一段很长的文本，如果分成两行而不是一行，"
                "会更容易阅读。"
    print(long_text)
```

你也可以跨行链式调用函数：

```mojo
def print_hello():
    text = ",".join("你好", " 世界！")
    print(text)
```

关于循环和条件语句的更多信息，请参阅[控制流](/docs/manual/control-flow/)。

## 函数

使用 `def` 关键字定义 Mojo 函数。例如，以下代码使用 `def` 关键字定义了一个名为 `greet` 的函数，该函数需要一个 `String` 参数并返回一个 `String`：

```mojo
def greet(name: String) -> String:
    return "你好，" + name + "！"
```

## 代码注释

使用井号 `#` 创建单行注释：

```mojo
# 这是注释。Mojo 编译器会忽略这一行。
```

注释也可以跟在代码后面：

```mojo
var message = "你好，世界！" # 这也是有效的注释
```

API 文档注释用三引号括起来。例如：

```mojo
def print(x: String):
    """打印字符串。

    Args:
        x: 要打印的字符串。
    """
    ...
```

使用这类注释（称为"文档字符串"）来记录你的代码是一个我们尚未完全规范的主题，但你可以使用 [`mojo doc` 命令](/docs/cli/doc/)从文档字符串生成 API 参考。

:::note

从技术上讲，文档字符串不是_注释_，而是 Mojo 多行字符串字面量语法的特殊用法。详情请参阅[类型](/docs/manual/types/)页面中的[字符串字面量](/docs/manual/types/#string-literals)。

:::

## 结构体

你可以将类型（或"对象"）的高级抽象构建为 `struct`。

Mojo 中的 `struct` 类似于 Python 中的 `class`：它们都支持方法、字段、运算符重载、元编程装饰器等。但是，Mojo 结构体是完全静态的——它们在编译时绑定，因此不允许动态分发或对结构进行任何运行时修改。（Mojo 未来也将支持 Python 风格的类。）

例如，这是一个基本的结构体：

```mojo
struct MyPair(Copyable):
    var first: Int
    var second: Int

    def __init__(out self, first: Int, second: Int):
        self.first = first
        self.second = second

    def __init__(out self, *, copy: Self):
        self.first = copy.first
        self.second = copy.second

    def dump(self):
        print(self.first, self.second)
```

使用方式如下：

```mojo
def use_mypair():
    var mine = MyPair(2, 4)
    mine.dump()
```

`MyPair` 结构体包含两个特殊方法：`__init__()` 构造函数和 `__init__(out self, *, copy: Self)` 拷贝构造函数。像这样的_生命周期方法_控制结构体的创建、复制、移动和销毁。

对于大多数简单类型，你不需要编写生命周期方法。你可以使用 `@fieldwise_init` 装饰器为你生成样板字段初始化器，如果你通过 trait 一致性请求，Mojo 会合成拷贝和移动构造函数。所以 `MyPair` 结构体可以简化为：

```mojo
@fieldwise_init
struct MyPair(Copyable):
    var first: Int
    var second: Int

    def dump(self):
        print(self.first, self.second)
```

更多详情请参阅[结构体](/docs/manual/structs/)页面。

### Trait（特征）

trait 就像是结构体的特征模板。如果你想创建一个具有 trait中定义的特征的结构体，你必须实现每个特征（如每个方法）。trait 中的每个特征都是结构体的"要求"，当你的结构体实现了所有要求时，就说它"符合"该 trait。

使用 trait 可以让你编写泛型函数，接受任何符合 trait 的类型，而不是只接受特定类型。

例如，创建 trait 的方式如下：

```mojo
trait SomeTrait:
    def required_method(self, x: Int): ...
```

方法签名后的三个点是 Mojo 语法，表示该方法未实现。

这是一个符合 `SomeTrait` 的结构体：

```mojo
@fieldwise_init
struct SomeStruct(SomeTrait):
    def required_method(self, x: Int):
        print("hello traits", x)
```

然后，这是一个使用 trait 作为参数类型（而不是结构体类型）的函数：

```mojo
def fun_with_traits[T: SomeTrait](x: T):
    x.required_method(42)

def use_trait_function():
    var thing = SomeStruct()
    fun_with_traits(thing)
```

你会看到 Mojo 标准库提供的很多 API 中都使用了 trait。例如，Mojo 的集合类型如 `List` 和 `Dict` 可以存储任何符合 `Copyable` trait 的类型。你可以在创建集合时指定类型：

```mojo
my_list = List[Float64]()
```

:::note

你可能想知道 `fun_with_traits()` 上的方括号。这些不是函数_参数_（参数放在圆括号中）；这些是函数_参数_，我们接下来会解释。

:::

如果没有 trait，`fun_with_traits()` 中的 `x` 参数就必须声明一个实现了 `required_method()` 的特定类型，比如 `SomeStruct`（但那样函数就只接受那个类型）。有了 trait，函数可以接受任何符合（即"实现"）`SomeTrait` 的类型作为 `x`。因此，`fun_with_traits()` 被称为"泛型函数"，因为它接受_泛化_类型而不是特定类型。

更多详情请参阅 [trait](/docs/manual/traits/) 页面。

## 参数化

在 Mojo 中，parameter（参数）是一个编译时变量，它成为运行时常量，在函数或结构体的方括号中声明。参数允许编译时元编程，这意味着你可以在编译时生成或修改代码。

许多其他语言互换使用"parameter"和"argument"，所以请注意，当我们说"parameter"和"参数化函数"时，我们指的是这些编译时参数。而函数"argument"（实参）是在圆括号中声明的运行时值。

参数化是一个复杂的话题，在[元编程](/docs/manual/parameters/)部分有更详细的介绍，但我们想在这里稍微介绍一下。作为入门，让我们看一个参数化函数：

```mojo
def repeat[count: Int](msg: String):
    # 在编译时评估以下 for 循环
    comptime for i in range(count):
        print(msg)
```

这个函数有一个 `Int` 类型的参数和一个 `String` 类型的实参。要调用该函数，你需要同时指定参数和实参：

```mojo
def call_repeat():
    repeat[3]("你好")
    # 打印 "你好" 3 次
```

通过将 `count` 指定为参数，Mojo 编译器能够优化函数，因为这个值保证在运行时不会改变。代码中的 `comptime` 关键字告诉编译器在编译时而不是运行时评估 `for` 循环。

编译器实际上生成了一个只重复消息 3 次的 `repeat()` 函数的唯一版本。这使得代码更高效，因为运行时需要计算的内容更少。

同样，你可以定义带参数的结构体，这实际上允许你在编译时根据参数值定义该类型的变体。

关于参数的更多详情，请参阅[元编程](/docs/manual/parameters/)部分。

## Python 集成

Mojo 支持原样导入 Python 模块，因此你可以立即利用现有的 Python 代码。

例如，导入和使用 NumPy 的方式如下：

```mojo
from std.python import Python

def main() raises:
    var np = Python.import_module("numpy")
    var ar = np.arange(15).reshape(3, 5)
    print(ar)
    print(ar.shape)
```

你必须在使用 Mojo 的环境中安装 Python 模块（如 `numpy`）。

更多详情请参阅 [Python 集成](/docs/manual/python/)页面。
